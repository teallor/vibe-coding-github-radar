# Changelog

All notable project changes will be documented here. This project follows semantic versioning where practical during its early stage.

## [Unreleased]

## [0.1.1] - 2026-07-06

### Added

- Scoring transparency documentation with evidence guidance and three evaluation templates.
- Operations and troubleshooting guidance for Actions, reports, Feishu, logs, and secrets.
- Public roadmap issues linked from the four-week maintenance plan.
- Three validated sample focus profiles with an explicit mapping to current runtime configuration.
- A credential-free Actions smoke-test step and documented failure categories.
- Structured error categories for GitHub, RSS, optional AI review, Feishu delivery, configuration, environment, and schedule/runtime failures.
- Independent, credential-free push and pull-request CI.

### Changed

- Improved troubleshooting documentation with machine-readable error codes and safe remediation guidance.
- Closed roadmap issue #3 after adding tests, structured diagnostics, and independent CI.
- Updated OpenAI Codex for Open Source evidence notes with verifiable releases, closed roadmap items, and CI/test evidence.

## [0.1.0] - 2026-07-06

### Added

- Initial public open-source discovery and learning workflow.
- Configurable project discovery, focus areas, keywords, filters, and scoring.
- Structured report generation and optional Feishu report delivery.
- Scheduled GitHub Actions automation and retained screening evidence.
- Automated tests for radar rules, reports, schedules, deduplication, delivery safeguards, and feedback adapters.
- Project governance, security, contribution, roadmap, release, and maintainer-workflow documentation.
