### 4. 小白名词解释

**GitHub**
- 是什么：一个存放代码的网站，全世界的人把项目放在上面共享
- 为什么需要：可以找到大量免费开源项目学习和使用
- 什么时候用：想找开源项目、托管自己的代码时
- 怎么判断是否用对：在上面搜索关键词，找到项目后阅读 README

**仓库 (Repository)**
- 是什么：GitHub 上一个项目的完整文件夹
- 为什么需要：是项目代码的存放地
- 什么时候用：访问项目时
- 怎么判断是否用对：看仓库名和 README 判断是否值得读

**README**
- 是什么：项目的说明书，告诉你这个项目是什么、怎么用
- 为什么需要：是了解项目的第一步
- 什么时候用：打开任何项目时先读
- 怎么判断是否用对：看 README 能不能让你快速理解项目

**License**
- 是什么：开源许可证，规定你能怎么使用这个项目的代码
- 为什么需要：避免侵权，知道能不能商用
- 什么时候用：想用别人代码时
- 怎么判断是否用对：MIT 最宽松，GPL 有传染性，无 License 需谨慎

**Star**
- 是什么：GitHub 上的点赞，代表有多少人觉得这个项目好
- 为什么需要：衡量项目受欢迎程度
- 什么时候用：判断项目质量时参考
- 怎么判断是否用对：Star 高说明受欢迎，但不一定适合你

**Fork**
- 是什么：把别人的项目复制一份到自己账号下
- 为什么需要：可以在自己的副本上修改
- 什么时候用：想基于别人项目改造时
- 怎么判断是否用对：Fork 后可以在自己仓库自由修改

**Issue**
- 是什么：项目的问题追踪，报告 bug 或提建议的地方
- 为什么需要：了解项目已知问题和活跃度
- 什么时候用：评估项目健康度时
- 怎么判断是否用对：Open Issues 多不一定差，看是否有人在回复

**Pull Request**
- 是什么：提交代码修改的请求
- 为什么需要：参与开源贡献的方式
- 什么时候用：你修改了代码想贡献回去时
- 怎么判断是否用对：提交 PR 后等待作者审核

**Commit**
- 是什么：一次代码提交，记录代码的变更
- 为什么需要：追踪每次改了什么
- 什么时候用：查看项目历史时
- 怎么判断是否用对：看 commit 历史了解项目演进

**API**
- 是什么：应用程序接口，程序之间对话的约定
- 为什么需要：让不同程序能互相调用功能
- 什么时候用：想用别人的功能时
- 怎么判断是否用对：看 API 文档了解怎么调用

**REST API**
- 是什么：一种常见的 API 设计风格
- 为什么需要：简单通用
- 什么时候用：做 Web 应用时
- 怎么判断是否用对：用 GET/POST/PUT/DELETE 操作资源

**Token**
- 是什么：一串密码一样的字符串，用来证明你有权访问
- 为什么需要：安全地访问需要认证的 API
- 什么时候用：调用需要登录的 API 时
- 怎么判断是否用对：绝对不能把 Token 放到前端代码里

**CLI**
- 是什么：命令行界面，在终端里用文字命令操作
- 为什么需要：高效，适合自动化
- 什么时候用：做脚本和自动化时
- 怎么判断是否用对：在终端输入命令运行

**依赖 (Dependency)**
- 是什么：项目运行需要的外部库或包
- 为什么需要：不用自己写所有功能
- 什么时候用：安装项目时
- 怎么判断是否用对：看 requirements.txt 或 package.json

**Python 包**
- 是什么：Python 的扩展模块
- 为什么需要：避免重复造轮子
- 什么时候用：需要某个功能时
- 怎么判断是否用对：用 pip install 安装

**配置文件**
- 是什么：存放项目设置参数的文件
- 为什么需要：不用改代码就能调整行为
- 什么时候用：修改项目设置时
- 怎么判断是否用对：看 .env、config.yaml、settings.json 等

**数据库**
- 是什么：有组织地存储数据的系统
- 为什么需要：高效存储和查询数据
- 什么时候用：需要持久化数据时
- 怎么判断是否用对：SQL 或 NoSQL，小项目可用 SQLite

**Markdown**
- 是什么：一种简单的排版语法
- 为什么需要：简单易学，适合写文档
- 什么时候用：写 README、笔记、报告时
- 怎么判断是否用对：用 # 标题、**加粗**、- 列表等语法

**HTML**
- 是什么：网页的骨架语言
- 为什么需要：所有网页的基础
- 什么时候用：做网页时
- 怎么判断是否用对：用标签定义内容

**本地运行**
- 是什么：在你自己的电脑上运行
- 为什么需要：免费、隐私、不依赖网络
- 什么时候用：个人工具优先本地运行
- 怎么判断是否用对：用 python 或 node 命令直接运行

**自动任务**
- 是什么：按计划自动执行的任务
- 为什么需要：节省重复劳动
- 什么时候用：需要定期执行的任务
- 怎么判断是否用对：用 GitHub Actions cron 或系统定时任务
